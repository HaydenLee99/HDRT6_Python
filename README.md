# DB vs pandas 성능 & 비용 비교 가이드

데이터 처리 시 **DB와 pandas 중 어디서 처리할지**에 대한 기준 정리

---

## 📊 비용 비교

| 항목 | 🗄️ DB에서 처리 | 🐼 pandas에서 처리 |
|------|---------------|-------------------|
| 🧠 CPU (연산) | 서버에서 최적화 처리 | Python 처리 (복잡하면 느림) |
| 💾 메모리 | DB 서버 사용 | 클라이언트 메모리 사용 ⚠️ |
| 🌐 네트워크 | 최소 데이터만 전송 ✔ | 전체 데이터 전송 ❌ |
| 🔁 쿼리 수 | 많으면 성능 저하 | 1회 조회 후 재사용 |
| 💿 I/O | 인덱스 기반 최적화 | 메모리 기반 |
| ⚡ 속도 (대용량) | 매우 빠름 🚀 | 느림 🐢 |
| ⚡ 속도 (소량) | 오버헤드 있음 | 빠름 |
| 🔒 동시성 / 락 | 영향 받음 | 영향 없음 |
| 🧩 유연성 | 제한적 | 매우 높음 👍 |
| 🛠️ 개발 생산성 | SQL 복잡 😵 | 코드 간단 😄 |
| 📈 시각화 | 거의 불가 | 매우 쉬움 🎨 |

---

## 🔑 핵심 포인트

```text
DB = 효율 (성능, 메모리, 네트워크)
pandas = 편의 (분석, 가공, 시각화)
